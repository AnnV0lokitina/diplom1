1) gophkeeper - клиентсткая часть, хранит клиентские файлы в file_store.
Текстовые и бинарные файлы будут хранится вмести с json файлом, который сожержит всю информацию пользователя.
Текстовые и бинарные файлы будут приходить и отправляться последовательно потоком через grpc (в зашифрованом виде)

2) service - серверная часть приложения. Имеет grpc api для регистрации, приема и передачи информации (файлы потоком
последовательно). Авторизация по логину, паролю. Сессии и данные авторизации хранятся в postgreSQL.
Зашифрованые данные пользователей будут храниться в file_store, где подпапка пользователя называется как id пользователя в БД.
